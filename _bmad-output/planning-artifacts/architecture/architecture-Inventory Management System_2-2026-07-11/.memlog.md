- (decision) AD-1: Partitioned local-first paradigm adopted. Edge devices write to local SQLite, central plane owns reconciliation. Degraded state visible on device.
- (decision) AD-2: Gate-token event chain. All inbound events linked by vehicle-to-PO binding token. Stitching by token, not timestamp.
- (decision) AD-3: DOA registry as single approval resolver. One enterprise registry; workflow config consumes, never overrides.
- (decision) AD-4: BOM as system of record for structure. ERP supplies cost rates inbound only. Conflicts create exceptions, never overwrites.
- (decision) AD-5: Production WIP distinct from R&D project WIP. Separate ledgers, separate stream types. No co-mingling.
- (decision) AD-6: Custody ledger non-valuated and segregated. Customer-owned stock in separate stream type. No closure with non-zero balance.
- (decision) AD-7: R&D stock flagged and blocked from cross-issue. Reclassification requires R&D-head approval.
- (decision) AD-8: Calibration lockout non-overridable. QC write handler checks calibration projection. No role can override rejection.
- (decision) AD-9: One asset register for everything. From two-tonne mould to hub screwdriver. Tool crib and custody cross-reference it.
- (decision) AD-10: Source-linked scrap intake only. Every scrap receipt must originate from a source document. Intake without source blocked.
- (decision) AD-11: Asset subledger posts to ERP GL. Platform is operational subledger, ERP is book of record. Depreciation previewed before posting.
- (decision) AD-12: Compliance spine as platform layer. Edit log, DOA, tagging, location, lockout, statutory triggers at bottom of dependency graph.
- (decision) AD-13: Nothing crosses the gate without a document. RGP/NRGP required. Gate enforcement by scan. No pass, no exit.
- (decision) AD-14: Read models are shared projections. Modules read from projections, not each other's event streams. No direct DB access.
- (decision) AD-15: Event-sourced location with asserted-vs-expected. Mismatches are exceptions, not silent overwrites. Location is a projection.
- (decision) AD-16: Idempotency keys on all edge-originated commands. Deduplication by task/source ID. HTTP 409 on duplicate.
- (version) Stack pinned: Node.js 24 LTS, PostgreSQL 18.4, PowerSync 1.23.x, Next.js 16 / TanStack Start 1.x, AWS Mumbai primary + Hyderabad DR.
- (decision) Event envelope defined: 11 fields including event_id, stream_type/id, event_type/version, payload JSONB, metadata with correlation_id, causation_id, actor, device_id, capture_method, occurred_at, synced_at, schema_version.
- (decision) Consistency conventions: UUIDs internal, _ext suffix for external IDs, UTC timestamps, business_date separate IST field, past-tense events, imperative commands, uniform error envelope.
- (decision) Deployment: ECS Fargate (not Kubernetes), RDS Aurora PostgreSQL Multi-AZ, CloudFront CDN. Three environments: dev, staging, production. DR in Hyderabad via Aurora Global Database.
- (decision) Retention: 8 financial years online for event store, cold storage in S3 Glacier, per-type retention classes for documents, DPDP crypto-shredding for PII erasure.
- (decision) API contract: REST over HTTPS, URL-versioned, SSO-gated, all mutating operations logged to edit log. Stable error codes defined.
- (decision) Spine acceptance contract: 5 testable contracts (edit log, DOA registry, event-sourced location, calibration lockout, business-stream tagging) as gate for Phase 1 module development.
- (event) Architecture spine finalized via party-mode session with PRD Room (John, Mary, Winston, Sally, Ravi). 16 ADs, 14 modules, 342 FRs mapped. Spine-first approach confirmed by sponsor.
